== AP

如果一台手机要通过wifi 加入交换机, 就需要借助 AP (Access Point，无线接入点),

[.my3]
[options="autowidth" cols="1a,1a"]
|===
|Header 1 |Header 2

|作用
|image:/img/0002.webp[,50%]

由于交换机通常只认识有线信号（以太网帧），而手机发送的是无线信号（无线电波），两者“语言不通”，必须通过 AP 进行转换。

#AP 的核心作用是将手机的 Wi-Fi 无线信号, 转换成"交换机"能够理解的有线网络信号。# 没有 AP，手机发出的信号交换机是“听不见”也“看不懂”的。

数据流向是这样的： +
手机（无线帧）​ →AP（接收并转换）​ →交换机（有线帧）​ →路由器/互联网

|你可能会有疑问：“我家没有单独买 AP，手机也能连 Wi-Fi 啊？”
|这是因为家用无线路由器, 其实是一个“三合一”设备： +
- AP：负责 Wi-Fi 功能。 +
- 交换机：负责 LAN 口的有线连接。 +
- 路由器：负责拨号上网和 NAT。 +

所以，当你手机连接家中的 Wi-Fi 时，你实际上是通过路由器内置的 "AP 模块", 接入了它内部的"交换机模块"。

*AP 就是让手机能够“无线”接入“有线交换机”的那个关键设备。​*

|企业级场景：独立 AP + 交换机
|但在企业、酒店、学校等大型网络中，为了覆盖更广、带机量更大，通常会采用独立的 AP：

== Router 路由器 -> 让"局域网"内的设备, 能访问"外网"(互联网)

[.my3]
[options="autowidth" cols="1a,1a"]
|===
|Header 1 |Header 2

|作用
|#路由器, 是用来连接不同网络的. 如, 从内网(家庭局域网)到外网(互联网), 就要通过路由器来连接.#


image:/img/0003.webp[,60%]

路由器则像是连接不同城市的交通枢纽。它的核心任务是##在不同网络之间（如从你家局域网到互联网），根据 IP 地址（逻辑地址）选择最佳路径，##并实现网络互连。

- #*当你访问Google时，数据包先发给"交换机"，"交换机"发现目标不在本地，就转给"路由器"，路由器再根据 IP 地址, 将其发送到互联网上.*#

通俗比喻：“跨城快递中转站”。它不关心大楼内部怎么送信，只关心这个包裹要发往北京还是上海，并决定走高速还是国道最快，同时负责把私人的门牌号（私有 IP）翻译成公共的收货地址（公网 IP）。

#*简单总结： +
-> "交换机"负责“打通内部”，让设备在"局域网"内高效对话； +
-> "路由器"负责“连接外部”，让局域网能访问"互联网"或"其他网络"。*#



|原理
|路由器通过"路由协议"（如 OSPF、BGP）#维护一张“网络地图”（路由表），根据 IP 地址决定数据包该走哪条路#（如走电信线路还是联通线路）才能到达目标网络。
|===

== LAN 口 (Local Area Network Port，局域网端口)

[.my3]
[options="autowidth" cols="1a,1a"]
|===
|Header 1 |Header 2

|功能
|##是设备用于连接"内部网络"的物理接口。##你可以把它理解为设备上的“内网专用网线插口”。

在"交换机"和"无线路由器"上，*这些端口专门负责将电脑、打印机、AP（无线接入点）等设备连接在一起，组建一个"内部局域网"（LAN）。*

- 内部互联：##**所有插在 LAN 口上的设备处于同一个“内网”，**##可以直接通信（如电脑访问网络打印机）。
- #*地址分配：通常由"路由器"通过 DHCP 服务, 给连接 LAN 口的设备, 分配私有 IP 地址（如 192.168.1.x）。*#

一句话总结：LAN 口就是设备上用来组建“内部小圈子”的网线插口，负责连接“自己人”。




|颜色/标识：
|在家用无线路由器上，#LAN 口通常用黄色标注#，并直接印有“LAN”字样，#与蓝色的“WAN/Internet”口区分开。#

image:/img/0004.jpg[,50%]

|交换机, 和路由器, 上的 LAN口区别
|- #*交换机: 它的所有端口本质上都是 LAN 口，只负责在"内部网络"里转发数据，没有“外网”概念。*# +
AP 通过网线插入路由器的 LAN 口，手机就相当于“虚拟地”接入了这个 LAN 口，从而成为局域网的一员。

- ##**路由器: 它的 LAN 口除了交换功能，还**##受"路由引擎"管理，#*负责将内网数据导向 WAN 口（互联网）。*#
|===

== MAC 地址 (Media Access Control) & IP 地址 (Internet Protocol)

核心区别：身份 vs 位置: +
MAC 地址是设备的“身份证号”，IP 地址是设备的“住址”。

image:/img/0005.webp[,%]

[.my1]
.案例
====
假设你要从北京的家（电脑A）, 给上海的公司（服务器B）寄一份文件（数据包）。 +

[.my3]
[options="autowidth" cols="1a,1a"]
|===
|Header 1 |Header 2

|#*IP 地址决定“大方向”（跨城路由）*#
|- ##你填写的收件人地址（上海市浦东新区XX路XX号）就是 目标 IP 地址。##它告诉快递系统（路由器）：“这个包裹最终要去上海”。 +
- ##你的寄件人地址（北京市海淀区XX小区）就是 源 IP 地址。##它告诉对方：“回信请寄回北京这个地址”。 +
- 路由器只关心 IP 地址，它根据 IP 决定包裹是走“京沪高速”还是“航空线路”。 +

|#*MAC 地址决定“小范围交付”（本地投递）*#
|- 快递员上门取件时，需要确认你的身份（扫描你的身份证），这个身份证号就是 源 MAC 地址。 +
- 快递员把包裹送到上海的公司楼下，前台签收时也要出示身份证登记，这个身份证号就是 目标 MAC 地址。 +
- #*交换机只关心 MAC 地址，它负责在“大楼内部”（局域网）把包裹准确送到具体工位。*# +
|===
